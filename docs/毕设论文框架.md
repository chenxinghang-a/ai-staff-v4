## 答辩策略

### 3个最可能被问的难题

**Q1：大模型存在"幻觉"且不可预测，如何保证在电气工程这种高安全要求的场景下不产生致命错误？**
- 承认局限 → 强调机制：V5质量门控闭环
- 双重校验：执行专家生成 → 审核专家对照GB/IEC标准审计 → 评分机制量化
- 人在回路(Human-in-the-loop)：定位为"辅助设计工具"，最终审核权在工程师

**Q2：你的"Skill上下文注入"与RAG/Function Calling有什么区别？为什么更好？**
- RAG/Function Calling是动态检索或调用，复杂长链路易丢失上下文
- 上下文注入是将工业库API规范作为"领域先验知识"在Prompt级别直接注入
- 优势：减少模型"何时调用"的犹豫，提高代码语法正确率，降低Token往返

**Q3：多智能体协作增加了复杂度和Token成本，是否具有经济合理性？**
- 分级路由：简单问题仅用低成本模型单次调用(12 token)
- 价值对冲：一次严重方案错误的损失远超几千Token成本，用少量Token增加换极高可靠性
